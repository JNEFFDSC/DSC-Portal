How Documents Are Delivered
When a client uploads a document, here's exactly how it flows to your team through your existing platforms.
SP
SharePoint / OneDrive
Files are uploaded via the portal and automatically routed to a structured SharePoint folder organized by deal. Each client gets their own subfolder: /Deals/[ClientName]-[PropertyAddr]/Docs/
1
Client drags & drops file into portal
2
Portal backend posts file to SharePoint via Microsoft Graph API using a service account
3
File lands in deal folder with metadata: doc type, uploader name, timestamp
4
SharePoint sends your team an email/Teams notification: "New document received for [Deal Name]"
SF
Salesforce
Each upload automatically updates the associated Opportunity record in Salesforce — marking the checklist item received, adding a timestamped Activity, and attaching the file as a Salesforce File/ContentDocument.
1
Portal identifies client's Salesforce Opportunity ID (passed via secure link or session)
2
Salesforce REST API call: update custom checkbox field "Promissory Note Received = true"
3
Activity/Task logged: "Document uploaded by client — [Doc Name] — [Timestamp]"
4
File attached as ContentDocument linked to the Opportunity and Contact records
5
Salesforce flow triggers email alert to assigned rep via RingCentral/email
DS
DocuSign
Once all required documents are received and reviewed, DocuSign envelopes are automatically generated for closing documents — Purchase & Sale Agreement, Assignment, Note Endorsement, and Payor/Insurance letters — and sent to the seller for eSignature.
1
Underwriter clicks "Send Closing Packet" once deal is approved
2
DocuSign API creates envelope with pre-tagged templates (W-9, PSA, Assignment, etc.)
3
Seller receives DocuSign email and signs electronically
4
Signed documents auto-save to SharePoint deal folder and Salesforce Opportunity
🔒
Security & Access
Each client accesses the portal via a unique, time-limited secure link generated by your CRM. Files are encrypted in transit (TLS 1.3) and at rest. No client can view another client's documents. All access is logged.
Implementation note: This portal is a frontend prototype. To activate the integrations above, a backend service (Node.js/Python) needs to be deployed to handle the API calls to Microsoft Graph, Salesforce REST API, and DocuSign eSign API. We can build that backend layer for you — ask us how.
